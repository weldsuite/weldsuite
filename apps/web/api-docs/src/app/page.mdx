## Getting started {{ anchor: false }}

The WeldSuite API is a REST API served under `https://api.weldsuite.org/v1`. It uses API key authentication with fine-grained permission scopes, no OAuth flows, just a `wsk_` key with exactly the permissions your integration needs. Every list endpoint is cursor-paginated and every response uses a consistent JSON envelope. {{ className: 'lead' }}

## Quick example

List people from your workspace:

```bash
curl https://api.weldsuite.org/v1/people \
  -H "Authorization: Bearer wsk_your_api_key"
```

```json
{
  "data": [
    {
      "id": "person_abc123",
      "firstName": "Jane",
      "lastName": "Doe",
      "fullName": "Jane Doe",
      "email": "jane@doe.example",
      "title": "Operations Director"
    }
  ],
  "pagination": {
    "totalCount": 42,
    "hasMore": false,
    "cursor": null
  }
}
```

## Available resources

The WeldSuite API exposes these product modules:

- **WeldCRM**, Companies, People, Leads, Opportunities, Activities, Quotes, Pipelines, Pipeline Stages
- **WeldFlow**, Projects, Tasks, Time Entries, Sprints, Milestones, Goals, Project Members, Project Files, Project Documents, Project Messages, Project Sheets, Project Labels, Task Comments, Task Tags, Whiteboards
- **WeldChat**, Channels, Channel Members, Messages, Drafts, Bookmarks, Sections
- **WeldCalendar**, Calendars, Events
- **WeldDrive**, Files, Folders, Drive (unified feed + storage stats)

Each resource is gated by its own permission scopes so you control exactly what your API keys can access.
